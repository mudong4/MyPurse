# 开发阶段规划 — 输出模板

> Agent 分析完需求文档后，按此模板输出阶段规划。

---

## 输出格式

### 第一步：功能拆解

```markdown
## 功能拆解

| 功能编号 | 功能名称 | 描述 | 依赖 | 复杂度 | 优先级 |
|----------|----------|------|------|--------|--------|
| F1 | xxx | xxx | 无 | 高/中/低 | P0/P1/P2 |
| F2 | xxx | xxx | F1 | 中 | P0 |
```

**依赖关系图**（Mermaid 或 ASCII）：

```
F0（基础设施）
├── F1（数据层完整实现）
│   ├── F2（核心页面组 A）
│   └── F3（核心页面组 B）
├── F4（辅助功能 C）
└── F5（数据管理）
```

### 第二步：阶段分组

```markdown
## 阶段规划

| 阶段 | 内容 | 预计工作量 | 验收标准 | 依赖 |
|------|------|------------|----------|------|
| 阶段 0 | 项目初始化 | 小 | xxx | 无 |
| 阶段 1 | 数据层基础 | 中 | xxx | 阶段 0 |
| 阶段 2 | xxx | 中 | xxx | 阶段 1 |
| ... | ... | ... | ... | ... |
| 阶段 N | 收尾优化（拆分为 Na/Nb/Nc） | 小/中 | xxx | 全部 |

**阶段总数**：N+1
**总预估工作量**：X 个阶段
```

### 第三步：逐阶段展开

每个阶段展开为详细表格：

```markdown
## 阶段 N：阶段名称

**状态**：🔲 待开发
**预计工作量**：小 / 中 / 大
**依赖**：阶段 N-1

| 事项 | 说明 | 验收方式 |
|------|------|----------|
| 具体任务 1 | 描述 | 如何验证完成 |
| 具体任务 2 | 描述 | 如何验证完成 |

### 本阶段新增文件（预估）
- `xxx/xxx.kt` — 功能描述

### 本阶段修改文件（预估）
- `xxx/xxx.kt` — 改动内容

### 本阶段测试要求
- [ ] 核心 DAO 单元测试
- [ ] UseCase 单元测试
- [ ] 手动验证验收标准全部通过

### 本阶段完成 Checklist
- [ ] 功能按验收标准正常工作
- [ ] 开发日志已追加
- [ ] 如有 Bug → 问题记录已更新
- [ ] 如有需求变更 → requirements.md 已更新
- [ ] 开发阶段.md 状态已标记为完成
```
