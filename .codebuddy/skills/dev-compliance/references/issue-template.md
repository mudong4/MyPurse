# 问题记录模板

追加到 `docs/3-开发过程/问题记录.md` 末尾时使用：

```markdown
---

### V1.0-#N：问题简短标题（会话 #N 修复/发现）

- **发现日期**：2026-MM-DD
- **严重程度**：🟠 功能缺陷 / 🟡 体验问题 / 🔴 崩溃
- **模块**：所属模块名
- **描述**：现象是什么
- **根因**：为什么发生
- **修复**：✅ 已修复（日期，会话号，一句话修法）
- **涉及文件**：`path/file.kt`
- **教训**：举一反三，以后怎么避免
```

精简版同步到 `docs/3-开发过程/问题记录-精简版.md`：
- 新增行到对应分类的表格
- 更新分类索引（如"颜色/样式问题"）
- 更新 V1.1 计划修复列表（如前移）
