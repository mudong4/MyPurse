## 九、经验教训总结

### 9.1 代码层面

| 教训 | 说明 |
|------|------|
| 接口拆分要配套集成 | 创建了不用等于死代码 |
| 分页要用合理数值 | Int.MAX_VALUE会导致OOM |
| 防御性编程要集成 | 光创建不用没意义 |
| 颜色等要统一配置 | 分散会导致不一致 |

### 9.2 规范层面

| 教训 | 说明 |
|------|------|
| Agent不会主动读规范 | 要提供必读清单 |
| 人类不知道审查什么 | 要提供审查要点 |
| 规范过于复杂不会被执行 | 要匹配项目规模 |
| 文档过长难以查阅 | 要有精简版 |

### 9.3 流程层面

| 教训 | 说明 |
|------|------|
| 重大修改要先讨论 | 防止擅自行动 |
| 编译验证是必须的 | 防止提交不可编译代码 |
| 文档更新要同步 | 防止信息不一致 |
| 历史记录要保留 | 方便追溯 |
