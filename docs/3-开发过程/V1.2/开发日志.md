### 2026-06-23 | 会话 #2：V1.2 设计讨论 — 隐患分析 + 删除选项简化

**作者**：CodeBuddy + 用户

**操作摘要**：
1. **隐患分析**：识别 V1.2-#1（`getTransactionById` 仍用快照名）和 V1.2-#2（SQLite GROUP BY 非严格模式边缘风险）
2. **快照名意义讨论**：确认快照名唯一不可替代的场景是「分类被删除后给历史记录兜底」
3. **删除选项简化决策**：删除分类弹窗从 3 选项减为 2 选项，移除「直接删除（保留记录）」。理由是孤儿数据无法参与统计筛选，对用户无实际价值

**修改文件**：

### 2026-06-23 | 会话 #3：V1.2 13b 合并分类迁移实现

**作者**：CodeBuddy

**操作摘要**：
1. **13b-1 DAO 层**：`TransactionDao` 新增 `updateCategoryL1Id(oldId, newId)` 和 `updateCategoryL2Id(oldId, newId)` 两个 UPDATE @Query
2. **13b-2 Repository**：`CategoryRepositoryImpl.mergeCategoryAndDelete()`：
   - 一级分类：迁移流水 L1 + 子分类按名称匹配归入目标（同名则流水合并、异名则移籍 parentId）+ 删源
   - 二级分类：直接 UPDATE L2 + 删源
3. **13b-3 ViewModel**：新增 `onMergeStart()` / `onMergeBack()` / `onMergeConfirm(targetId)` 三步流程；移除 `onDeleteKeepRecords` 和 `onBatchDeleteKeepRecords`
4. **13b-3 UI**：
   - 单个删除弹窗 `CategoryDeleteDialog`：2 选项（①删除及记录 ②合并到其他分类），选②进入目标选择步骤显示候选分类列表
   - 批量删除弹窗 `BatchDeleteDialog`：简化为单确认按钮
