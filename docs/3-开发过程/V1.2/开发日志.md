### 2026-06-23 | 会话 #7：移除流水分页

**作者**：CodeBuddy + 用户

**操作摘要**：
1. 用户决策：个人记账数据量极少超出单次加载上限，分页无实际价值，反而增加代码复杂度
2. 移除分页逻辑：
   - ViewModel：删除 `currentOffset`、`pageSize`、`accumulatedList`、`loadMore()`；`loadData()` 移除 `append` 参数；5 个加载方法 + 搜索方法改为一次性全量加载
   - UiState：移除 `hasMore` 字段
   - Screen：移除 `shouldLoadMore` 计算、`loadMore()` LaunchedEffect、底部加载 spinner、`isLoadingMore` 参数
3. 各视图独立上限：日500 / 周1000 / 月3000 / 分类3000 / 年10000
4. 修复编译错误：首次批量 `hasMore` 移除遗漏 `loadCategoryView` 中 1 处 + 其余 4 个加载方法中 4 处残留

**修改文件**：

| 文件 | 变更内容 |
|---|---|
| `ui/transactionlist/TransactionListViewModel.kt` | 删除分页字段/方法，所有加载方法简化为一次性全量加载，各视图独立上限 |
| `ui/transactionlist/TransactionListUiState.kt` | 移除 `hasMore` 字段 |
| `ui/transactionlist/TransactionListScreen.kt` | 移除 `derivedStateOf`、`shouldLoadMore`、`LaunchedEffect(loadMore)`、底部 spinner、`isLoadingMore` 参数 |
