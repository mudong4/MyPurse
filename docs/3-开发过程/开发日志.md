**关键设计决策**：
- **防抖方案**：选择 Compose 层面 `rememberDebounce()` 而非 ViewModel 全局标志位。理由：①每个 Composable 独立防抖状态，互不干扰；②不引入额外 StateFlow 状态管理复杂度；③防抖窗口 500ms 足够防止双击但不影响正常操作节奏
- **数据库异常兜底**：不在 `build()` 层做全局 try-catch（Room 构建失败是单例级别的致命错误），而是捕获 `SQLiteDatabaseCorruptException` 和 `SQLiteCantOpenDatabaseException` 两种可恢复异常，自动删除损坏文件并重建。种子数据由 `DatabaseInitializer` 在首次查询时自动播种，用户无感知
- **冷启动白屏**：使用 `windowBackground` 主题色方案（兼容 API 24+），而非 Android 12+ SplashScreen API（后者只覆盖 API 31+）。`#F8F9FA` 与 Compose 背景色一致
- **StrictMode**：仅在 debug 构建启用，检测主线程磁盘读写和网络调用。release 构建零开销
- **废弃代码确认**：`FlowTypeRepository` 及其空实现方法是 V1.1 预留扩展点，注释清晰，不属于废弃代码，保留
- **边界防御确认**：代码层面极端金额（0.01/9,999,999.99）、空列表（availableYears→WheelPicker）、跨年 BETWEEN 查询、amount 为空不提交等均防御充分
