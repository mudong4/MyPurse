## 问题列表

### 问题 #1：分类管理拖拽排序性能卡顿

- **发现日期**：2026-06-15
- **严重程度**：🟡 体验问题（PC 端已可感知，移动端预期更严重）
- **所属模块**：分类管理页 (E)

**现象**：
拖拽分类条目移动时，画面有明显卡顿，不够跟手。

**根因分析**：
1. `onDrag` 回调每帧（~60fps）都会调用 `onMoveCategory`，触发 LazyColumn 重组和逐条数据库写入
2. `animateDpAsState` 为每个列表项创建动画状态对象，增加重组开销
3. 每次拖拽过程中逐条执行 `updateSortOrder`，产生大量数据库 I/O

**解决方案**：
1. `onDrag` 中仅累积 `dragOffsetY`，不做任何数据变更；实际的位置交换逻辑移至 `onDragEnd` 执行
2. 将 `animateDpAsState` 替换为简单的条件判断（`if (isDragging) 8.dp else 0.dp`）
3. 新增 `batchUpdateSortOrder` DAO 方法，使用 `@Transaction` 在单次事务中批量更新所有受影响分类的排序序号

**涉及文件**：
- `CategoryManageScreen.kt`：拖拽手势逻辑重构
- `CategoryManageViewModel.kt`：`onMoveCategory` 改为调用批量更新
- `CategoryDefDao.kt`：新增 `batchUpdateSortOrder`
- `CategoryRepository.kt` / `CategoryRepositoryImpl.kt`：新增批量更新接口及实现

### 问题 #2：拖拽排序后点击位置错乱

- **发现日期**：2026-06-15
- **严重程度**：🟠 功能缺陷
- **所属模块**：分类管理页 (E)

**现象**：
拖拽"餐饮"到新位置后，再次点击"餐饮"，响应的是原来位置（拖拽前的位置）的分类。

**根因分析**：
1. **闭包捕获了旧值**：`LazyColumn` 中 `itemsIndexed` 的 `pointerInput(category.id)` 的 key 未因重排序而改变，导致手势检测回调中捕获的 `index` 和 `categories` 仍是拖拽前的旧值
2. **UI 状态被覆盖**：`loadCategories()` 中的 `collect` 使用了 `_uiState.value.copy(...)` 而非 `current.copy(...)`，导致 `expandedParentId` 等 UI 状态字段在数据刷新时被重置为默认值

**解决方案**：
1. 在 `itemsIndexed` 的 lambda 中使用 `rememberUpdatedState` 包装 `index` 和 `categories`，确保长生命周期的 gesture 回调始终读取最新值
2. 将 `_uiState.value.copy(...)` 改为 `current.copy(...)`，保留所有 UI 状态字段

**涉及文件**：
- `CategoryManageScreen.kt`：添加 `rememberUpdatedState`，修复手势回调中的闭包捕获
- `CategoryManageViewModel.kt`：修复 `loadCategories()` 中的状态合并方式

**教训**：
- Compose 中 `pointerInput` 的 key 不变时，内部 lambda 捕获的变量不会自动更新，必须用 `rememberUpdatedState` 桥接
- `StateFlow.collect` 中使用 `value.copy()` 会丢弃 collect 之前的并发更新，应使用 `current.copy()`

### 问题 #3："恢复默认"按钮无响应

- **发现日期**：2026-06-15
- **严重程度**：🟠 功能缺陷
- **所属模块**：分类管理页 (E)

**现象**：
1. 点击"恢复默认"按钮后，分类顺序未恢复，UI 无变化
2. 按钮仅恢复支出侧还是同时恢复收入侧不明确

**根因分析**：
1. `restoreDefaultCategories` 方法只处理了"添加缺失的一级分类"，完全忽略了：
   - 已存在分类的 `sortOrder` 恢复为默认值
   - 子分类的恢复
2. 子分类数据（`CategoryDefaults.subCategories`）从未被使用

**解决方案**：
重写 `restoreDefaultCategories` 方法，执行以下三步：
1. 将已存在的一级分类的 `sortOrder` 重置为默认值
2. 添加缺失的一级分类（含正确的 `sortOrder`）
3. 遍历 `CategoryDefaults.subCategories`，添加缺失的子分类

**涉及文件**：
- `CategoryRepositoryImpl.kt`：重写 `restoreDefaultCategories`

**待确认**：
- 用户提问"恢复默认是把收入和支出都恢复吗"——当前实现仅恢复当前选中的那一侧（支出或收入），通过顶部 Tab 切换后需分别点击恢复。后续可考虑增加"恢复全部"选项。
