## 1. 项目概览

**MyPurse** 是一款个人记账 Android 应用，纯本地运行（无后端），以「随手记」为核心体验。

| 维度 | 详情 |
|---|---|
| 语言 | Kotlin 2.0.21 |
| UI 框架 | Jetpack Compose + Material 3 |
| 最低 SDK | 24 (Android 7.0) |
| 目标 SDK | 35 |
| 架构 | MVVM + Clean Architecture (Data/Domain/UI 三层) |
| 依赖注入 | Hilt (Dagger) |
| 数据库 | Room (SQLite) |
| 导航 | Jetpack Navigation Compose (类型安全路由) |
| 序列化 | kotlinx.serialization (路由参数) |
| 构建工具 | Gradle 8.7.3 + KSP |

## 3. 架构设计

### 3.1 三层架构

```
┌──────────────────────────────────────┐
│  UI 层 (ui/)                         │
│  Screen + ViewModel + UiState        │  ← Jetpack Compose
│  依赖 → domain 层                    │
├──────────────────────────────────────┤
│  领域层 (domain/)                    │
│  Model + Repository接口 + UseCase    │  ← 纯 Kotlin，不依赖 Android
│  依赖 → 无（只依赖 Kotlin 标准库）    │
├──────────────────────────────────────┤
│  数据层 (data/)                      │
│  Repository实现 + DAO + Entity       │  ← Room + Hilt
│  实现 domain 层接口                   │
└──────────────────────────────────────┘
```

**关键原则**：
- 依赖方向：UI → Domain ← Data（Domain 是核心，不依赖任何外层）
- 数据层实现 Domain 层接口（依赖倒置）
- Entity 仅存在于 data 层，UI 层只接触 Domain Model

### 3.2 数据流

```
用户操作 → Screen (Composable)
              ↓ 调用方法
         ViewModel (Hilt 注入)
              ↓ 调用 Repository 接口
         RepositoryImpl (data 层)
              ↓ 调用 DAO
         Room Database (SQLite)
```

ViewModel 通过 `StateFlow<UiState>` 向下传递状态，Screen 通过 `collectAsState()` 订阅。

### 3.3 导航设计

使用 Jetpack Navigation Compose + 类型安全路由 (`@Serializable sealed class Route`)：

| 路由 | 对应页面 | 参数 |
|---|---|---|
| `Route.Home` | 首页 | 无 |
| `Route.Statistics` | 统计页 | 无 |
| `Route.Settings` | 设置页 | 无 |
| `Route.AddTransaction(flowType, date)` | 记一笔 | flowType, date (可选) |
| `Route.TransactionList(flowType)` | 流水列表 | flowType (可选) |
| `Route.TransactionDetail(id)` | 详情页 | id: Long |
| `Route.CategoryManage` | 分类管理 | 无 |
| `Route.Budget` | 预算设置 | 无 |
| `Route.RecurringTemplateList` | 模板列表 | 无 (推迟) |
| `Route.RecurringTemplateEdit(id)` | 模板编辑 | id (可选) (推迟) |
| `Route.About` | 关于页 | 无 |

## 5. 关键技术决策与注意点

### 5.1 金额存储

- 使用 `BigDecimal`，Room 通过 `BigDecimalConverter` 存为 TEXT
- **绝对不要用 Float/Double**，会产生精度问题

### 5.2 日期处理

- 数据库存储为 Unix 毫秒时间戳 (`Long`)
- CSV 导出格式为 `yyyy-MM-dd`
- 统计页时间范围计算需注意时区，统一用 `Calendar` + 手动归零时分秒

### 5.3 WheelPicker 组件

项目中有多处滚轮选择器，需特别注意索引偏移问题：

- `ChineseDatePickerDialog`（记一笔 + 流水列表共用）中的 WheelPicker
- `StatisticsScreen` 中的 `YearSheet` / `YearMonthSheet` 有独立的 WheelPicker 副本

**已知坑**：`LazyColumn` 中有 spacer item 时，`selectedIndex` 到 `firstVisibleIndex` 的映射必须精确计算。spacerCount 定义为 1 时：
```
全局索引 = spacerCount + selectedIndex
居中偏移 = spacerCount + selectedIndex - visibleItems/2
```
修改 WheelPicker 时，需同步检查统计页的副本。

### 5.4 空列表安全

异步加载的列表数据初始可能为空 `[]`。调用 `coerceIn(0, items.lastIndex)` 时若 `items` 为空会 crash。必须先判空：

```kotlin
if (items.isEmpty()) return
val safeIndex = index.coerceIn(0, items.lastIndex)
```

### 5.5 分类管理级联删除

删除分类时需级联处理关联流水：`CategoryRepositoryImpl` 中先查关联流水，更新或删除后再删分类。当前策略是将关联流水的分类设为 null。

### 5.6 CSV 导入导出

- 导出：全量导出为 UTF-8 BOM CSV（确保 Excel 打开不乱码），通过 FileProvider 分享
- 导入：SAF 文件选择 → 表头校验 → 逐条校验 → 自动创建不存在的分类 → 追加写入
- CSV 表头：`流水类型,一级分类,二级分类,金额,备注,日期`

### 5.7 预算极简方案

V1.0 采用极简方案：`budget` 表仅维护一条记录 (id=1)，通过 upsert 更新。如需支持多月预算，需重新设计表结构。

## 7. 常见修改场景指引

### 7.1 添加新页面

1. 在 `Route.kt` 添加新路由对象
2. 在 `MyPurseNavHost.kt` 添加 `composable<Route.Xxx>` 块
3. 在 `ui/` 下创建新目录，含 Screen + ViewModel + UiState
4. 如需新数据，在 `domain/repository/` 加接口方法，`data/repository/` 实现
5. 在 `di/RepositoryModule.kt` 中绑定（如新增 Repository）

### 7.2 修改数据库

1. 修改 Entity 文件
2. 修改对应 DAO
3. **递增 `@Database` 的 version**
4. 添加 `AutoMigration` 或实现 `Migration`
5. 注意：Room 2.6.1 不支持 `@DeleteTable` 等自动迁移注解，需手写 SQL

### 7.3 添加新依赖

1. 在 `gradle/libs.versions.toml` 的 `[versions]` 和 `[libraries]` 中添加
2. 在 `app/build.gradle.kts` 的 `dependencies {}` 中引用
3. 如需 KSP/KAPT 处理，添加对应配置

### 7.4 修改内置分类种子数据

编辑 `data/repository/CategoryDefaults.kt`。修改后：
- 新安装用户自动获得新分类
- 老用户需手动「恢复默认分类」或清除数据后重新播种

## 9. 版本管理（Git）

### 9.1 当前状态

- 分支：`master`
- 远程：`origin/master`
- 最新 Tag：`v1.0`（V1.0 发布版本）
- 所有阶段完成提交在 master 上

### 9.2 发布构建

**前置条件**：
- JDK 17+
- 签名密钥文件 `mypurse-release.jks` 位于项目根目录（已通过 `.gitignore` 排除）

**构建命令**：
```bash
# Debug 构建
./gradlew assembleDebug

# Release 构建（含混淆+资源压缩）
./gradlew assembleRelease
```

**Release APK 输出路径**：`app/build/outputs/apk/release/app-release.apk`

**⚠️ 必须保留的文件**（构建后）：
- `app/build/outputs/mapping/release/mapping.txt` — 混淆映射文件，用于还原线上崩溃堆栈

**签名密钥管理**：
- 密钥文件：`mypurse-release.jks`（RSA 2048，有效期 10000 天）
- Alias：`mypurse-release`
- 密码当前硬编码在 `build.gradle.kts` 中，正式分发时建议改为环境变量注入

### 9.2 版本回退方法

如果更新某个功能后发现不想要或有问题：

**方法一：回退文件（推荐，保留其他修改）**
```bash
# 回退单个文件到上次提交
git checkout -- <文件路径>

# 回退整个目录
git checkout -- app/src/main/java/com/wyd/mypurse/ui/settings/
```

**方法二：回退整个提交（保留工作区）**
```bash
# 撤销最近一次提交，修改回到工作区
git reset --soft HEAD~1
```

**方法三：回退并丢弃修改**
```bash
# 撤销最近一次提交并丢弃所有修改
git reset --hard HEAD~1
```

**方法四：查看历史找到某个版本**
```bash
# 查看提交历史
git log --oneline

# 查看某个提交改了哪些文件
git show <commit-hash> --stat

# 临时切换到某个历史版本查看（不影响当前分支）
git checkout <commit-hash>
```

**方法五：从远程恢复**
```bash
# 如果本地搞乱了，从远程强制恢复
git fetch origin
git reset --hard origin/master
```

### 9.3 安全操作建议

- 大改动前先 `git commit` 保存当前状态
- 用 `git stash` 暂存未提交的修改
- 不确定时先 `git log` 看清楚
- `git reset --hard` 不可逆，谨慎使用
