## 目录结构

```
docs/
├── README.md                         ← 本文件：文档索引入口
├── 1-项目概览/
│   └── 项目介绍.md                    ← 架构、技术栈、安全修改指南
├── 2-需求规格/
│   ├── requirements.md               ← V1.0 封版功能规格、交互设计、变更记录
│   └── requirements-v1x.md           ← V1.x 迭代需求（优化清单、修复计划）
├── 3-开发过程/
│   ├── 开发阶段.md                    ← 当前进度、各阶段内容
│   ├── 开发日志.md                    ← 每次会话的完整改动记录
│   ├── 开发日志-精简版.md            ← 日志速查表
│   ├── 问题记录.md                    ← Bug 案例、根因、教训（完整版）
│   └── 问题记录-精简版.md            ← 问题速查表
├── 4-规范与指南/
│   ├── 项目启动指南.md                ← 需求澄清 + 文档体系初始化
│   ├── 开发阶段规划指南.md            ← 如何根据需求文档设计阶段
│   ├── 通用开发规范.md                ← 通用方法论（可复用到其他项目）
│   └── MyPurse开发规范.md              ← 项目特定编码规范（Room、Compose、BigDecimal）
└── 5-优化建议/
    ├── V1.0回退后待修复清单.md        ← V1.0 发布后发现的待修复项
    └── 项目管理优化方案.md            ← 文档/流程层面的优化建议
```

## Agent 接手项目时的阅读顺序

```
1. 1-项目概览/项目介绍.md       → 了解项目全貌、架构、关键决策
2. 3-开发过程/开发日志-精简版.md  → 快速了解开发历程（详细内容读完整版）
3. 3-开发过程/问题记录-精简版.md  → 快速了解已知坑点（详细内容读完整版）
4. 2-需求规格/requirements.md    → 了解 V1.0 功能规格和交互设计
5. 2-需求规格/requirements-v1x.md → 了解 V1.x 迭代需求和修复计划
6. 4-规范与指南/MyPurse开发规范.md → 了解项目特定编码约束（重点：2.3、11、14.6 节）
7. 3-开发过程/开发阶段.md        → 了解当前进度、待完成事项
8. 4-规范与指南/通用开发规范.md  → 了解通用方法论（重点：0.3 节）
```

## Agent 完成任务后必须做的事

> 详细规则见 `4-规范与指南/MyPurse开发规范.md` 13.2 节。以下为精简版必检清单：

```
□ 1. 开发日志已追加本次会话记录（含修改文件清单 + 设计决策）
□ 2. 开发日志-精简版已同步更新
□ 3. 如有 Bug 修复 → 问题记录已更新，问题记录-精简版已同步
□ 4. 开发阶段.md 中涉及的子阶段项状态列已打勾 ✅/🔲
□ 5. 如有需求/设计变更 → requirements 正文 + 第10章变更记录已更新
□ 6. 临时文件已清理（编译输出、调试日志等）
```

## 文档表格速查

| 我想知道... | 读这个 |
|-------------|--------|
| 项目整体怎么设计的？怎么安全修改？ | `1-项目概览/项目介绍.md` |
| V1.0 功能规格是什么？需求怎么变的？ | `2-需求规格/requirements.md` |
| V1.x 迭代需求和修复计划 | `2-需求规格/requirements-v1x.md` |
| 当前进度到哪了？每个阶段包含什么？ | `3-开发过程/开发阶段.md` |
| 某天做了什么？改了哪些文件？（快速查阅） | `3-开发过程/开发日志-精简版.md` → `开发日志.md` |
| 遇到过什么 Bug？怎么解决的？（快速查阅） | `3-开发过程/问题记录-精简版.md` → `问题记录.md` |
| 编码、架构、Git、AI 协作的强制规范 | `4-规范与指南/MyPurse开发规范.md` |
| 通用方法论（可复用到其他项目） | `4-规范与指南/通用开发规范.md` |
