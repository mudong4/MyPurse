# MyPurse 文档索引

> **文档版本**：V1.1 (2026-06-17)

> **给 Agent 的阅读顺序**：从 1 到 4，依次阅读。

---

## 目录结构

```
docs/
├── README.md                         ← 本文件：文档索引入口
├── 1-项目概览/
│   └── 项目介绍.md                    ← 架构、技术栈、安全修改指南
├── 2-需求规格/
│   └── requirements.md               ← 功能规格、交互设计、变更记录
├── 3-开发过程/
│   ├── 开发阶段.md                    ← 当前进度、各阶段内容
│   ├── 开发日志.md                    ← 每次会话的改动记录
│   └── 问题记录.md                    ← Bug 案例、根因、教训
└── 4-规范与指南/
    ├── 项目启动指南.md                ← 需求澄清 + 文档体系初始化（给 Agent 用）
    ├── 开发阶段规划指南.md            ← 如何根据需求文档设计阶段（给 Agent 用）
    ├── 通用开发规范.md                ← 通用方法论（可复用到其他项目）
    └── MyPurse开发规范.md              ← 项目特定编码规范（Room、Compose、BigDecimal）
```

## Agent 接手项目时的阅读顺序

```
1. 1-项目概览/项目介绍.md       → 了解项目全貌、架构、关键决策
2. 3-开发过程/开发日志.md        → 了解最近做了什么、改了哪些文件
3. 3-开发过程/问题记录.md        → 了解已知 Bug 和坑点（避免重复踩坑）
4. 2-需求规格/requirements.md    → 了解功能规格和交互设计
5. 4-规范与指南/MyPurse开发规范.md → 了解项目特定编码约束
6. 3-开发过程/开发阶段.md        → 了解当前进度、待完成事项
7. 4-规范与指南/通用开发规范.md  → 了解通用方法论
8. 4-规范与指南/项目启动指南.md    → 如需从零启动新项目，参考此指南
9. 4-规范与指南/开发阶段规划指南.md → 如需规划新阶段，参考此指南
```

## Agent 完成任务后必须做的事

1. 更新 `3-开发过程/开发日志.md`（追加本次改动记录）
2. 如有 Bug 修复 → 更新 `3-开发过程/问题记录.md`
3. 如阶段完成 → 更新 `3-开发过程/开发阶段.md`
4. 如需求变更 → 更新 `2-需求规格/requirements.md`
5. 如架构调整 → 更新 `1-项目概览/项目介绍.md`
6. 删除所有临时文件（编译输出、调试日志等）
