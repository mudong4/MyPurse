# 个人记账 App 需求规格说明书（V1.0 封版）

## 1. 项目背景与产品定位

**背景**  
现有主流记账软件功能臃肿、启动慢、广告多，个人日常记账场景中大量功能未被使用。调研「随手记」后发现其核心体验良好，但启动广告和复杂功能令人困扰。

**产品定位**  
一款**极致轻量、零广告、纯本地、启动即记**的个人记账工具，专注为晚间回顾记账场景提供流畅的连续补记体验，同时兼顾消费后即时记录。固定周期自动记账计划于 V1.1 实现。架构上预留联网与 AI 扩展点。

**核心价值**

- 打开即用，无广告干扰
    
- 晚间可连续补记多条流水，流程顺畅不中断
    
- 完全本地存储，零权限申请
    
- 灵活可配置的两级分类（支出/收入独立管理）
    
- 多维度流水筛选与双视图统计（构成分析 + 趋势分析）
    
- 极简预算管控（单一值），首页实时进度提醒
    
- 固定周期收支自动记录 <!-- 推迟到 V1.1 -->
    
- 代码结构预留未来云同步与 AI 分析扩展点
    

---

## 2. 目标用户与使用场景

**用户画像**  
普通上班族，习惯在晚上花几分钟集中补记当日开销，偶尔也需在消费后即时记录。

**使用场景与权重**

|场景|占比|描述|核心需求|
|---|---|---|---|
|晚间回顾记账|75%|每天睡前花 3-5 分钟，回想当日消费，逐条补记。|连续快速添加多条记录，操作不中断，默认值合理，减少重复选择。|
|消费后即时记账|15%|买完咖啡、吃完午饭等消费后，立即掏出手机记录。|从打开 App 到保存完成路径极短，交互直接。|
|自动记账|10%|房租、订阅会员等固定收支，设置一次后自动记录。|设置简单，自动执行可靠，不重复记账。<!-- 推迟到 V1.1 -->|

## 3. 功能性需求（MVP V1.0）

### 3.1 记一笔模块 `(V1.0)`

|功能点|详细描述|
|---|---|
|流水类型|V1.0 仅内置「支出」和「收入」两种，不开放自定义（日常场景已足够）。切换流水类型时，分类列表自动切换为对应侧的支出分类或收入分类。预留接口供后续版本扩展。|

### 3.2 分类管理模块 `(V1.0)`

> **设计原则**：支出和收入的分类完全隔离，各自独立管理。分类管理页通过顶部 Tab 切换「支出分类」和「收入分类」，两套分类互不干扰。每个分类通过 `flowSign` 字段标记归属（-1=支出侧，1=收入侧）。

|功能点|详细描述|
|---|---|
|支出侧内置分类|一级：餐饮、交通、购物、娱乐、住房、通讯、医疗、教育、其他支出。每个一级下预设二级细项。内置分类**允许删除**。|
|收入侧内置分类|一级：工资、奖金、理财、兼职、报销、退款、红包、其他收入。每个一级下预设二级细项。内置分类**允许删除**。|
|自定义分类|用户可新增一级和二级分类，修改名称，更换图标，排序（拖拽）。新增时需指定归属（支出侧/收入侧）。|
|恢复默认|分类管理页提供"恢复默认分类"按钮，一键还原当前侧的全部内置分类（已存在的跳过，不重复添加）。|
|修改/删除规则|- 修改分类名称：仅影响后续选择，历史记录保留快照名称。  <br>- 删除分类：弹出三个选项：①直接删除（历史保留原分类快照名，ID 关联失效）②迁移至其他分类（选择目标分类，历史记录批量替换 ID 和快照名）③删除分类及所有关联记录。  <br>- 无分类时记一笔：弹出引导提示"还没有分类，去添加一个吧"，并自动跳转到分类管理页。|
|批量修改分类|提供「一键修改分类」工具，可按原分类名（含已删除分类）批量替换为新分类，支持按时间范围过滤，预览影响条目数后执行，操作放入一个数据库事务中。|
|批量删除分类|分类管理页顶部栏进入批量模式后，多选一级分类（选中父分类时子分类级联），确认弹窗二选一：保留历史记录 / 删除分类及关联记录。|

### 3.4 统计模块 `(V1.0)`

#### 3.4.1 整体设计思路

统计模块回答用户的两个核心问题，对应**两个子视图**，用户点击 Tab 切换：

|子视图|回答的问题|主力图表|适用时间粒度|
|---|---|---|---|
|**构成分析**|钱花在哪些分类？|横向条形图（排名）|日/周/月/季/年|
|**趋势分析**|支出随时间怎么变化？|柱状图 + 折线图叠加|日/周/月/季/年|

> **设计决策（2026-06-16）**：砍掉"对比分析"子视图。构成分析 + 趋势分析已能覆盖用户核心诉求（"花在哪"+"怎么变"），对比分析信息增量有限但开发和交互复杂度高，ROI 低。如后续有用户反馈再考虑。

#### 3.4.2 时间维度选择器（统一控制两个子视图）

- 位于统计页顶部，是两个子视图共享的总开关。
    
- 粒度切换方式：点击当前时间标签，弹出粒度选择菜单（日/周/月/季/年）。
    
- 各粒度下的显示与切换：
    

|粒度|默认值|显示格式|切换控件|
|---|---|---|---|
|按天|今天|"6月7日 周六"|左右箭头逐天切换；点击日期弹出日历|
|按周|本周|"6月1日 - 6月7日"|左右箭头逐周切换；点击弹出周选择器|
|按月|本月|"2026年6月"|左右箭头逐月切换；点击弹出年月选择器|
|按季|本季|"2026年Q2 (4-6月)"|左右箭头逐季切换；点击弹出季选择器|
|按年|今年|"2026年"|左右箭头逐年切换；点击弹出年份选择器|

- 两个子视图共享同一时间选择器，切换粒度时两个视图同步更新。
    
- 统计页顶部显示概览卡片：当前时间范围的**支出总额**和**收入总额**（两项）。

> **设计决策（2026-06-16）**：概览卡片不显示预算剩余。预算的核心场景是"控制当月花销"，属于首页的前瞻性指标；统计页是回顾分析，加入预算指标语义不匹配。且预算数据依赖尚未实现的预算模块，不应阻塞统计阶段。
    

#### 3.4.3 子视图一：构成分析

**目的**：一眼看清当前时间范围内，钱主要花在哪些分类。

|区域|描述|
|---|---|
|支出/收入 Tab|顶部切换查看支出构成或收入构成。|
|横向条形图（主力）|各一级分类按金额从高到低排列，每个分类一个颜色，条形长度代表金额，右侧显示金额 + 占比%。底部显示合计。已删除分类用灰色默认图标。|
|二级分类明细|点击某条一级分类后，下方展开该分类的二级构成（同样是横向条形图，按金额从高到低）。再次点击同一分类收起。|
|流水跳转入口|点击"查看该分类全部流水 >"跳转到流水列表页，自动筛选为该分类 + 当前时间范围。|
|饼图切换按钮|右上角提供饼图切换按钮，用户可按需切换为饼图视图（辅助）。饼图仅显示 TOP5 分类，其余合并为"其他"，避免扇区过多不可读。|

#### 3.4.4 子视图二：趋势分析

**目的**：看清支出随时间的变化趋势，发现异常月份，判断消费习惯是否改善。

**粒度与图表类型**：

|当前粒度|趋势图 X 轴|数据点数量|图表类型|说明|
|---|---|---|---|---|
|按天|过去 30 天|30 个点|**纯折线图**|30 根柱子太密集，折线图更适合连续密集数据点|
|按周|过去 12 周|12 个点|柱状图 + 折线叠加|12 根柱子宽度合适|
|按月（默认）|过去 12 个月|12 个点|柱状图 + 折线叠加|12 根柱子宽度合适|
|按季|过去 8 个季度|8 个点|柱状图 + 折线叠加|数据点较少，柱子更宽|
|按年|全部年份|全部|柱状图|年份数量不定，通常不多|

> **设计决策（2026-06-16）**：按天粒度下柱子太密（每根仅约 10dp），改用纯折线图。X 轴标签在日粒度下每 5 天显示一个标签（1号、6号、11号...），避免标签重叠。

|区域|描述|
|---|---|
|图表模式切换|两个按钮：纯柱状 / 柱+折叠加。默认叠加模式。按天粒度下仅显示折线图，无切换按钮。|
|图表区|显示过去 N 个时间单位的支出变化。**不显示预算线**（统计页不做预算展示）。|
|数据摘要|图表下方自动计算并显示：均值、最高月份及金额、最低月份及金额、环比变化。|

#### 3.4.5 数据呈现原则

1. **颜色语义**：支出一律暖色系（不同分类不同色相），收入冷色系。
    
2. **排序原则**：所有分类排名图一律从高到低。
    
3. **数字精确**：金额显示到分（¥3,280.50），占比显示一位小数（32.5%）。支出金额显示红色负数（-¥30.00），收入显示绿色正数（+¥5,000.00）。
    
4. **空状态**：该时间范围无记录时，显示"暂无记账数据"并提供快捷记一笔入口。
    
5. **性能**：统计计算用 SQL 聚合（SUM + GROUP BY）在数据库层完成，不做内存遍历。
    

#### 3.4.6 技术实现要点

- 图表库：Vico（Compose 原生图表库）
    
- 时间范围变化时，ViewModel 重新查询数据并更新 StateFlow，图表自动重组
    
- 按日/周/月/季/年的分组逻辑封装在 `GetStatisticsUseCase` 中
    
- 所有聚合查询尽量用 Room 的 `@Query` 在 SQL 层完成
    
- 首页趋势图（`TrendChartCard`）和 TOP5（`TopCategoriesCard`）保留，作为进入统计页的快捷入口，与统计页内容独立、不冲突
    

### 3.5 预算模块 `(V1.0)`

#### 3.5.1 概念说明

预算与结余是两个独立概念：

|概念|定义|计算方式|
|---|---|---|
|**结余**|账面上还剩多少钱|从记账日到现在的全部收入 - 全部支出|
|**预算**|本月计划花多少钱的上限|用户手动设定的金额目标|

#### 3.5.2 存储模型（极简方案）

预算采用**单一值**存储于 budget 表，表中永远只有一条记录（id=1）。

|字段|类型|说明|
|---|---|---|
|id|Int|固定为 1（主键）|
|amount|BigDecimal|月度预算金额|
|updatedAt|Long|最后修改时间（毫秒时间戳）|

> **设计决策（2026-06-16）**：从原按月独立存储（year+month 复合主键 + isAutoGenerated）简化为单一值。
> 理由：个人记账场景下，历史预算值几乎不会被回顾；自动沿用、本月/长期生效切换等逻辑复杂度高但收益极低。
> 极简方案零额外逻辑，一个输入框即可。如需扩展（保留历史、分类预算），可通过 Room Migration 平滑升级，改动局限在数据层。

#### 3.5.3 默认行为

- 新用户首次使用：**不设预算**。首页预算进度条显示引导文字"设置月度预算 >"。
- 用户设置预算后：立即生效，全局唯一预算值。
- 用户修改预算：覆盖原值。

#### 3.5.4 预算功能在各页面的体现

|页面|体现|形式|
|---|---|---|
|**首页**|预算进度条|进度条 + "已花 ¥x / 预算 ¥x" + "剩余 ¥x"。正常蓝色，≥80% 橙色，超支红色。无预算时显示引导文字。点击跳转预算设置页。|
|**首页**|预算设置入口|顶部栏右侧日历图标按钮，点击跳转预算设置页。|
|**预算设置页**|预算配置|金额输入 + 快捷金额按钮 + 保存。|

#### 3.5.5 预算设置页面

|区域|描述|
|---|---|
|金额输入|数字键盘输入预算金额，¥ 前缀，占位符 0.00。|
|快捷设置|3 个预设金额按钮（1000/5000/10000），点击快速填入。每个按钮旁有编辑图标，点击可自定义该快捷金额值。|
|保存按钮|保存预算，成功后弹出确认弹窗。金额为 0 时按钮置灰。|
|不设预算|已设预算时显示红色"不设预算"按钮，点击清除预算值。|

### 3.6 数据管理 `(V1.0)`

|功能点|详细描述|
|---|---|
|导出 CSV|支持导出全部记录或按时间范围筛选后的记录，包含：流水类型、一级分类、二级分类、金额、备注、日期。文件通过系统分享菜单发送到公共存储（如下载目录）。|
|导入 CSV|选择 CSV 文件，校验格式，追加导入不覆盖。导入完成后反馈结果（成功 X 条 / 跳过 X 条 / 错误 X 条）。导入时自动创建不存在的分类。|
|数据存储|完全本地 Room 数据库，数据存放在应用内部存储。|
|清除所有数据|二次确认后删除所有数据（流水、分类、预算、模板），内置分类重新加载。|
|卸载行为|卸载 App 时，系统自动删除所有内部存储数据（含数据库），无残留。用户主动导出到公共存储的 CSV 文件不会被删除。|
|权限|V1.0 为零权限应用。导入 CSV 使用系统文件选择器（SAF），无需额外权限。|

#### 3.6.1 CSV 格式规范

**文件编码**：UTF-8 with BOM（确保 Excel 直接打开不乱码）

**表头（第一行）**：
```
流水类型,一级分类,二级分类,金额,备注,日期
```

**数据行格式**：
- 分隔符：逗号 `,`
- 文本字段（含逗号、换行、双引号时）用双引号包裹，内部双引号转义为 `""`
- 日期格式：`yyyy-MM-dd`（如 `2026-06-17`）
- 金额格式：纯数字，两位小数（如 `30.00`、`5000.00`），不含货币符号和千分位
- 流水类型：`支出` 或 `收入`

**示例**：
```csv
流水类型,一级分类,二级分类,金额,备注,日期
支出,餐饮,午餐,30.00,公司楼下食堂,2026-06-17
支出,交通,地铁,8.00,,2026-06-16
收入,工资,月薪,15000.00,6月工资,2026-06-15
支出,购物,衣服,299.00,"尺码偏大,换了一次",2026-06-14
```

**校验规则**：
- 第一行必须是标准表头（顺序可不同，但字段名必须完全匹配）
- 流水类型只能为 `支出` 或 `收入`
- 金额必须是有效正数（大于 0）
- 日期格式必须为 `yyyy-MM-dd` 且为有效日期
- 一级分类不能为空
- 校验失败的行**跳过**，不影响其他行导入

### 3.7 固定收支自动记账

> **推迟到 V1.1**。理由：场景占比仅 10%，时间周期计算复杂度高、易出 bug，手动记账成本极低。详见第 10 章变更记录。

#### 3.7.1 典型场景

|场景|示例|周期|
|---|---|---|
|每月固定支出|房租、房贷、物业费、订阅会员|每月 X 日|
|每周固定支出|每周五聚餐、每周一交通充值|每周 X|
|每年固定支出|保险年费、域名续费、会员年费|每年 X 月 X 日|
|每日固定支出|每日早餐预算、通勤费|每天|

#### 3.7.2 触发与执行

- **触发时机**：每次 App 启动时检查一次（V1.0 不依赖后台服务）。
    
- **执行逻辑**：
    
    1. 查询所有启用的模板
        
    2. 对每个模板，检查 `nextExecutionDate ≤ 今天` 的应执行日期
        
    3. 对每个应执行日期，检查该日期是否已有同一模板的记录，无则自动创建一条收支记录
        
    4. 更新 `lastExecutedDate` 和 `nextExecutionDate`
        
- **补记策略**：若错过多个周期，一次性补记所有遗漏周期，最多补记过去 90 天内。**V1.0 限制**：自动记账仅在 App 启动时触发，不依赖后台服务。若用户连续超过 90 天未打开 App，超出部分将不会被补记。V1.1 计划通过通知提醒改善此问题。
    
- **防重复**：自动创建的记录关联 `recurringTemplateId`，执行前检查该日期是否已有同一模板的记录。
    
- **无效日期处理**：如每月 31 日，当月无 31 日时自动退到当月最后一天。
    

#### 3.7.3 与流水列表的联动

- 自动生成的记录在流水列表中显示 🔄 标记
    
- 编辑自动记录后，`recurringTemplateId` 置为 null，变为手动记录
    
- 删除自动记录时，提示模板仍会在下次到期时生成新记录

## 7. 扩展性设计（联网与 AI 预留）

### 7.1 架构预留

- **数据仓库接口**：`TransactionRepository` 为接口，当前实现为 `LocalTransactionRepositoryImpl`，未来可添加 `RemoteTransactionRepositoryImpl` 实现同步。
    
- **同步管理器**：`sync` 包中定义 `SyncManager` 接口，V1.0 为空实现。
    
- **用户账户占位**：`UserManager` 返回本地默认用户，后续可接入云登录。
    

### 7.2 AI 接入设想

- 预留 `AnalyticsEngine` 接口，V1.0 使用本地规则实现预算提醒，未来可替换为远程 AI 服务。
    
- 潜在 AI 功能：智能分类建议、自然语言记账（"午餐30元"）、消费习惯预测、异常消费提醒。
    
- 数据层面已预留完整字段，后期仅需网络层发送数据并处理回调，需用户授权与隐私合规。
    

### 7.3 多账本扩展

- 数据库已含 `ledgerId` 字段，V1.0 默认值 "default"，V1.2 版本将支持创建多个账本并切换。

## 9. 验收标准（V1.0）

1. 应用安装后不申请任何权限即可正常使用全部功能。
    
2. 中等配置手机冷启动时间 ≤ 1 秒。
    
3. 即时记账完整操作 ≤ 5 秒。
    
4. 连续补记单笔间隔 ≤ 8 秒。
    
5. 分类管理页提供"恢复默认分类"按钮，正确还原内置分类。
    
6. 删除分类时可选择三个选项：直接删除（保留记录）/ 迁移至其他分类 / 删除分类及记录。批量修改分类功能可正确预览并执行。
    
7. 无分类时记一笔弹出引导并跳转到分类管理页。
    
8. 流水列表支持日/周/月/年视图切换，年视图摘要模式；分类筛选和搜索可组合使用，切换粒度时分类筛选保持。空状态正确显示。自动记录显示 🔄 标记。
    
9. 统计页支持日/周/月/季/年时间粒度切换，构成分析与趋势分析两个子视图同步更新。
    
10. 构成分析：横向条形图按金额从高到低排列，支持展开二级分类和切换饼图。已删除分类用灰色图标显示。
    
11. 趋势分析：支持纯柱状/柱+折叠加两种图表模式，按天粒度下为纯折线图。

12. 预算为单一值存储，修改即覆盖。首页进度条颜色和文字正确反映预算状态。
    
14. 固定模板支持每天/每周/每月/每年，启动时自动补记（最多 90 天），不重复生成。<!-- 推迟到 V1.1 -->
    
15. 编辑自动生成的记录解除模板关联；删除自动记录弹出特定提示。<!-- 推迟到 V1.1 -->
    
16. CSV 导入/导出字段齐全，校验正确，导入自动创建不存在的分类。
    
17. 所有金额使用 BigDecimal 精确计算，无浮点误差。
    
18. 至少 10,000 条记录下，列表滑动无卡顿，统计图表加载在 1 秒内。
    
19. 卸载 App 后重新安装，数据为空（确认无残留）。
